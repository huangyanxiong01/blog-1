1. 在使用别的工具进行调试之前，都要先确定是否能在以下两个地方获取到有用的信息
  - 执行画面上
  - 自定义日志
  - /var/log/syslog、/var/log/message、dmesg
2. 根据 CPU 使用率和进程状态判断故障类型
  - 高 CPU (几乎 100%)、进程状态 R     =>   某处陷入死循环
  - 低 CPU (几乎 0)、进程状态 S 或 D   =>   死锁、睡眠、等待信号
  - CPU 0、进程状态 T                 =>   SIGSTOP 信号导致
3. GDB 发现死循环
4. 记得开 core dump
